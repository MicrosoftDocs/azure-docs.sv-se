## <a name="planning-and-implementation"></a>Planering och implementering
Rubrik: SAP NetWeaver på Azure Virtual Machines – planerings-och implementerings guide

Sammanfattning: Starta med det här dokumentet om du funderar på att köra SAP NetWeaver i virtuella Azure-datorer. Den här planerings- och implementeringsguiden hjälper dig att utvärdera om ett befintligt eller planerat SAP NetWeaver-baserat system kan distribueras till en miljö för virtuella Azure-datorer. Den omfattar flera SAP NetWeaver distributionsscenarier och innehåller SAP-konfigurationer som är specifika för Azure. Pappret listar och beskriver all nödvändig konfigurationsinformation som du behöver på SAP/Azure-sidan för att köra ett SAP-hybridlandskap. Åtgärder som du kan vidta för att garantera hög tillgänglighet för SAP NetWeaver-baserade system på IaaS omfattas också.

## <a name="deployment"></a>Distribution
Rubrik: SAP NetWeaver på Azure Virtual Machines – distributions guide

Sammanfattning: Det här dokumentet innehåller riktlinjer för distribution av programvara för SAP NetWeaver till virtuella datorer i Azure. Det här dokumentet fokuserar på tre specifika distributionsscenarier med betoning på att aktivera Azure Monitoring-tillägg för SAP, inklusive felsökningsrekommendationer för Azure Monitoring-tillägget för SAP. Det här dokumentet förutsätter att du har läst planerings- och implementeringsguiden.

## <a name="sap-dbms-on-azure"></a>SAP-DBMS på Azure
Rubrik: SAP-DBMS i distributions guiden för Azure

Sammanfattning: Det här dokumentet beskriver planerings- och implementeringsöverväganden för DBMS system som ska köras tillsammans med SAP. I den första delen listas och presenteras allmänna överväganden. Följande delar av papperet avser distributioner av olika DBMS i Azure som stöds av SAP. Olika DBMS som presenteras är SQL Server, SAP ASE, Oracle, SAP MaxDB och IBM DB2 för Linux, UNIX och Windows. I de specifika delarna diskuteras de överväganden du behöver beakta när du kör SAP system på Azure tillsammans med de olika DBMS. Ämnen som vilka metoder för säkerhetskopiering och hög tillgänglighet som stöds av olika DBMS på Azure presenteras för användning med SAP-program.

Uppdaterad: december 2015

## <a name="sap-netweaver-on-azure"></a>SAP NetWeaver på Azure
Rubrik: SAP NetWeaver – skapa en Azure-baserad katastrof återställnings lösning

Sammanfattning: det här dokumentet innehåller en steg-för-steg-vägledning för att skapa en Azure-baserad katastrof återställnings lösning för SAP NetWeaver. Den lösning som beskrivs förutsätter att SAP-landskapet kör virtualiserad lokalt baserat på Hyper-V. I den första delen av dokumentet Azure Site Recovery-tjänster (ASR) introduceras i dess komponenter. Den andra delen av dokumentet beskriver information om SAP NetWeaver-baserade landskap. Möjligheter att använda ASR med SAP NetWeaver-programinstanser och centrala SAP-tjänster diskuteras och beskrivs. En fokusering på den andra delen använder ASR för centrala SAP-tjänster som skyddas med konfigurationer för Windows Server-redundanskluster.

Uppdaterad: september 2015
