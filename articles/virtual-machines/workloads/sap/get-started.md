# <a name="use-azure-to-host-and-run-sap-workload-scenarios"></a>Använda Azure som värd för och köra SAP-arbetsbelastningsscenarier

När du använder Microsoft Azure kan du tillförlitligt köra dina verksamhetskritiska SAP-arbetsbelastningar och -scenarier på en skalbar, kompatibel och företagsbeprövad plattform. Du får skalbarhet, flexibilitet och kostnadsbesparingar i Azure. Med det utökade samarbetet mellan Microsoft och SAP kan du köra SAP-program i utvecklings- och test- och produktionsscenarier i Azure och få fullständigt stöd. Vi har gått in på allt från SAP NetWeaver till SAP S/4HANA, SAP BI på Linux till Windows och SAP HANA till SQL.

Förutom att vara värd för SAP NetWeaver-scenarier med olika DBMS i Azure kan du vara värd för andra SAP-arbetsbelastningsscenarier, till exempel SAP BI på Azure. 

Det unika med Azure for SAP HANA är ett erbjudande som skiljer Azure från varandra. För att kunna vara värd för mer minne och CPU-resurskrävande SAP-scenarier som involverar SAP HANA, erbjuder Azure användning av kunddedikerad bare metal-maskinvara. Använd den här lösningen för att köra SAP HANA-distributioner som kräver upp till 24 TB (120 TB skalning) minne för S/4HANA eller annan SAP HANA arbetsbelastning. 

Att vara värd för SAP-arbetsbelastningsscenarier i Azure kan också skapa krav för identitetsintegrering och enkel inloggning. Den här situationen kan uppstå när du använder Azure Active Directory (Azure AD) för att ansluta olika SAP-komponenter och SAP SaaS-erbjudanden (programvara som en tjänst) eller PaaS-erbjudanden (plattform som en tjänst). En lista över sådana integrerings- och enkel inloggningsscenarier med Azure AD- och SAP-entiteter beskrivs och dokumenteras i avsnittet "Azure AD SAP identity integration and single sign-on" (Azure AD SAP-identitetsintegrering och enkel inloggning).
