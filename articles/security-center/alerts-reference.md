# <a name="security-alerts---a-reference-guide"></a>Säkerhetsaviseringar – en referensguide

Den här artikeln innehåller de säkerhetsaviseringar som du kan få Azure Security Center alla Azure Defender som du har aktiverat. Aviseringarna som visas i din miljö beror på vilka resurser och tjänster du skyddar, samt din anpassade konfiguration.

Längst ned på den här sidan finns en tabell som beskriver Azure Security Center kill-kedjan justerad med version 7 av [MITRE ATT&CK-matrisen](https://attack.mitre.org/versions/v7/).

[Lär dig hur du svarar på dessa aviseringar.](security-center-managing-and-responding-alerts.md)

[Lär dig hur du exporterar aviseringar.](continuous-export.md)

## <a name="mitre-attck-tactics"></a>MITRE ATT&CK-taktiker <a name="intentions"></a>

Att förstå avsikten med en attack kan hjälpa dig att undersöka och rapportera händelsen enklare. För att hjälpa till med dessa åtgärder Azure Security Center aviseringarna MITRE-taktiker med många aviseringar.

Den serie steg som beskriver förloppet för en cyberattack från rekognosering till data exfiltrering kallas ofta för en "händelsekedja". 

Security Center avsikter för händelsekedjan som stöds baseras på version 7 av [MITRE ATT&CK-matrisen](https://attack.mitre.org/versions/v7/) och beskrivs i tabellen nedan.

| Taktik                  | Beskrivning                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
|-------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **PreAttack**           | PreAttack kan antingen vara ett försök att få åtkomst till en viss resurs oavsett en skadlig avsikt eller ett misslyckat försök att få åtkomst till ett målsystem för att samla in information före utnyttjande. Det här steget identifieras vanligtvis som ett försök att, som kommer från utanför nätverket, genomsöka målsystemet och identifiera en startpunkt.                                                                                                                                |
| **InitialAccess**       | InitialAccess är den fas där en angripare lyckas få ett fäste på den angripna resursen. Det här steget är relevant för beräkningsvärdar och resurser som användarkonton, certifikat osv. Hot aktörer kommer ofta att kunna kontrollera resursen efter det här steget. |

Hot aktörer behöver ofta upprätthålla åtkomst till system genom avbrott, till exempel omstarter av systemet, förlust av autentiseringsuppgifter eller andra fel som kräver ett verktyg för fjärråtkomst för att starta om eller ge en alternativ bakdörr för att få åtkomst.                                                                                                                                                                                            |
